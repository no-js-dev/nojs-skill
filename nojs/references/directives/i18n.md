# Internationalization Directives

Translation directives for multi-language apps. `t` has priority 20; `i18n-ns` has priority 1.

## Contents

- [t](#t) -- translate element content
- [t-*](#t-1) -- pass parameters to translations

- [$i18n Context Variable](#i18n-context-variable) -- reactive proxy with dot-notation translation access

## Syntax Reference

| Directive | Syntax | Description |
|-----------|--------|-------------|
| `t` | `t="key"` | Translate element content by key |
| `t-*` | `t-name="expression"` | Pass interpolation parameter to translation |
| `t-html` | `t-html` | Render translation as sanitized HTML |
| `i18n-ns` | `i18n-ns="namespace"` | Set i18n namespace for element and descendants |

---

## `t`

Translate element content using an i18n key.

**Syntax:** `<element t="namespace.key">`

```html
<h1 t="landing.hero.title"></h1>
<a route="/" t="nav.home"></a>
```

### Edge Cases

- If the key is not found, the key string itself is rendered as the text content.
- Keys are dot-separated for nested lookup: `t="section.subsection.key"`.
- When the locale changes, all `t` elements automatically re-render with the new locale's translation.

### Complete Example

```html
<script>
  NoJS.i18n({
    defaultLocale: 'en',
    locales: {
      en: { nav: { home: 'Home', about: 'About', contact: 'Contact' } },
      pt: { nav: { home: 'Inicio', about: 'Sobre', contact: 'Contato' } }
    }
  });
</script>

<nav>
  <a route="/" t="nav.home"></a>
  <a route="/about" t="nav.about"></a>
  <a route="/contact" t="nav.contact"></a>
</nav>

<button on:click="$i18n.locale = 'pt'">Portugues</button>
<button on:click="$i18n.locale = 'en'">English</button>
```

## `t-*`

Pass parameters to translation strings.

**Syntax:** `<element t="key" t-paramName="expression">`

Parameters are injected into the translation string using `{paramName}` placeholders. Also supports pluralization via the pipe `|` separator in translation values.

```html
<!-- Translation: "Hello, {name}!" -->
<h1 t="greeting" t-name="user.name"></h1>

<!-- Pluralization: "{count} item | {count} items" -->
<span t="items" t-count="cart.items.length"></span>
```

### Edge Cases

- Pluralization uses the pipe `|` separator: `"one form | plural form"`. The first form is used when `count` is 1; the second for all other values.
- Multiple parameters can be combined: `<span t="message" t-name="user.name" t-count="items.length">`.
- Parameters are reactive -- changing the expression value re-renders the translation.

### Complete Example

```html
<script>
  NoJS.i18n({
    defaultLocale: 'en',
    locales: {
      en: {
        greeting: 'Hello, {name}!',
        items: '{count} item | {count} items',
        welcome: 'Welcome back, {name}. You have {count} notification | Welcome back, {name}. You have {count} notifications'
      }
    }
  });
</script>

<div state="{ user: { name: 'Erick' }, notifications: 3 }">
  <h1 t="greeting" t-name="user.name"></h1>
  <p t="welcome" t-name="user.name" t-count="notifications"></p>
  <span t="items" t-count="notifications"></span>
</div>
```

## `t-html`

Render the translation value as sanitized HTML instead of plain text. Companion to `t`.

**Syntax:** `<element t="key" t-html>`

```html
<!-- Translation: "Read our <a href='/terms'>terms</a>" -->
<div t="legal.notice" t-html></div>
```

The output is sanitized via `_sanitizeHtml()` to prevent XSS.

### Edge Cases

- Without `t-html`, HTML tags in the translation value are rendered as literal text.
- Script tags and event handlers in the translation are stripped during sanitization.

### Complete Example

```html
<script>
  NoJS.i18n({
    defaultLocale: 'en',
    locales: {
      en: {
        legal: 'By signing up, you agree to our <a href="/terms">Terms</a> and <a href="/privacy">Privacy Policy</a>.',
        highlight: 'This is <strong>very important</strong> information.'
      }
    }
  });
</script>

<p t="legal" t-html></p>
<p t="highlight" t-html></p>
```

## `i18n-ns`

Set the i18n namespace for an element and its descendants. Triggers loading of the namespace JSON file.

**Syntax:** `<element i18n-ns="namespace">` or `<element i18n-ns>` (auto-derive from route)

```html
<!-- Explicit namespace -->
<div i18n-ns="settings">
  <h2 t="settings.title"></h2>
</div>

<!-- Auto-derive on route-view -->
<main route-view src="templates/" route-index="landing" i18n-ns></main>
```

### Edge Cases

- When `i18n-ns` is used without a value on `route-view`, the namespace is auto-derived from the route filename.
- Namespaces are loaded on demand. The first time a namespace is encountered, the corresponding JSON file is fetched.
- When the locale is switched, all previously loaded namespaces (including route-loaded ones) are automatically re-fetched for the new locale.

## i18n Configuration

### Inline Locales

```html
<script>
  NoJS.i18n({
    defaultLocale: 'en',
    fallbackLocale: 'en',
    locales: {
      en: {
        greeting: 'Hello, {name}!',
        items: '{count} item | {count} items'
      },
      'pt-BR': {
        greeting: 'Ola, {name}!',
        items: '{count} item | {count} itens'
      }
    }
  });
</script>
```

### External Locale Files (Flat Mode)

```html
<script>
  NoJS.i18n({
    defaultLocale: 'en',
    loadPath: '/locales/{locale}.json'
  });
</script>
```

### Namespace Mode

Split translations by feature, loaded on demand:

```html
<script>
  NoJS.i18n({
    defaultLocale: 'en',
    loadPath: '/locales/{locale}/{ns}.json',
    ns: ['common']  // namespaces to load at startup
  });
</script>
```

Folder structure:

```text
locales/
  en/
    common.json
    settings.json
    dashboard.json
  pt-BR/
    common.json
    settings.json
    dashboard.json
```

### Caching

Locale files are cached in memory by default. Disable for development:

```html
<script>
  NoJS.i18n({ cache: false });
</script>
```

### Persistence

Persist the selected locale across page reloads:

```html
<script>
  NoJS.i18n({ persist: true });
</script>
```

## Locale Switching

Switch the active locale at runtime:

```html
<button on:click="$i18n.locale = 'pt-BR'">Portugues</button>
<span bind="$i18n.locale"></span>
```

All `t` elements, `class-*` expressions referencing `$i18n`, and namespace files automatically update when the locale changes.

---

## `$i18n` Context Variable

`$i18n` is a reactive Proxy that exposes translations as dot-notation properties in any expression context. When the locale changes, all bindings that reference `$i18n` paths automatically re-render.

### Properties

| Property | Description |
|----------|-------------|
| `$i18n.[namespace].[key]` | Translation string via dot-notation path (e.g. `$i18n.common.buttons.save`) |
| `$i18n.locale` | Current active locale (get/set) |
| `$i18n.locales` | Array of available locale codes |
| `$i18n.t(key, params)` | Translate a key with interpolation/pluralization |
| `$i18n.setLocale(code)` | Change the active locale programmatically |

Reserved properties (`locale`, `locales`, `t`, `setLocale`) are not resolved as translation keys.

### Dot-Notation Access

Access any translation key directly as a property path on `$i18n`:

```html
<!-- Direct binding to a translation string -->
<span bind="$i18n.shell.sidebar.intro"></span>

<!-- In state initialization -->
<div state="{ label: $i18n.common.title }">
  <h2 bind="label"></h2>
</div>

<!-- In bind -->
<button bind="$i18n.common.buttons.save" on:click="save()"></button>

<!-- Interpolation and pluralization via .t() -->
<span bind="$i18n.t('items', { count: cart.length })"></span>

<!-- Change locale -->
<button on:click="$i18n.setLocale('pt')">Portugues</button>

<!-- Display available locales -->
<span each="loc in $i18n.locales" bind="loc"></span>
```

### Expression Contexts

`$i18n.[path]` works in all expression contexts:

| Context | Example |
|---------|---------|
| `state` | `state="{ label: $i18n.common.title }"` |
| `store` | `value="{ heading: $i18n.common.heading }"` |
| `computed` | `expr="$i18n.common.greeting + ' ' + name"` |
| `watch` | `watch="$i18n.locale"` |
| `foreach` | `each="item in $i18n.nav.items"` |
| `bind` | `bind="$i18n.common.buttons.save"` |
| `if`/`show`/`hide` | `if="$i18n.locale === 'en'"` |

### Complete Example

```html
<script>
  NoJS.i18n({
    defaultLocale: 'en',
    loadPath: '/locales/{locale}/{ns}.json',
    ns: ['common']
  });
</script>

<div state="{ heading: $i18n.common.welcome }">
  <h1 bind="heading"></h1>
  <p bind="$i18n.common.description"></p>

  <nav>
    <a route="/" bind="$i18n.common.nav.home"></a>
    <a route="/about" bind="$i18n.common.nav.about"></a>
  </nav>

  <p bind="$i18n.t('common.items', { count: 5 })"></p>

  <div>
    <span>Current locale: <span bind="$i18n.locale"></span></span>
    <button on:click="$i18n.setLocale('en')">English</button>
    <button on:click="$i18n.setLocale('pt')">Portugues</button>
  </div>
</div>
```

## Number and Date Formatting

Locale-aware formatting filters using `Intl` APIs:

```html
<span bind="price | currency"></span>          <!-- $1,234.56 -->
<span bind="price | currency:'BRL'"></span>    <!-- R$ 1.234,56 -->
<span bind="total | number"></span>            <!-- 1,234.56 -->
<span bind="ratio | percent"></span>           <!-- 85% -->
<span bind="createdAt | date"></span>          <!-- Mar 23, 2026 -->
<span bind="createdAt | date:'long'"></span>   <!-- March 23, 2026 -->
<span bind="createdAt | datetime"></span>      <!-- Mar 23, 2026, 2:30 PM -->
<span bind="updatedAt | relative"></span>      <!-- 2 hours ago -->
```

These filters respect the current `$i18n.locale` and produce locale-appropriate output.
